大家好，我是齐昂！

昨天有一个新闻，关注技术的朋友应该都知道，就是 OpenClaw 超越 React、Linux，登顶 GitHub 开源第一！

然后今天我要说的 Tabbit，是美团光年之外团队这两天发布的一款 AI 浏览器。

对，又是一款 AI 浏览器， "又要在我电脑上吃灰了"，在我未体验之前我是这样想的。

毕竟我电脑上现在已经装了 10 个浏览器，有自带的 Safari，有微软的 Edge，然后各种 AI 浏览器：Arc，Dia，Commet，ChatGPT AtLas，Fellou，flowithOS，Quark。

当然还有依然坚挺，我使用最多的Chrome （Chrome前段时间也出了 Gemini in Chrome, 我开启了下，由于地域原因，没能正常打开就没折腾使用了，不做评价）。    

现在除了Chrome我经常使用，Commet，Dia，Arc 偶尔使用，其他浏览器基本都是吃灰了。

这四个的比例大概是 65:20:10:5。

但是在用了 Tabbit 之后, 这个比例可能会是 Tabbit : Others = 80 : 20 了。

接下来我来分享下我的使用体验。

先简单介绍下 Tabbit，

Tabbit 分：
- [国际版](https://www.tabbitbrowser.com/) 
- [国内版](https://www.tabbit-ai.com/)

最大的区别可能就是能使用的模型了,国际版支持国外三巨头+国内模型的海外部署版本，而国内版则只能使用国内模型。

| 版本 | 支持的模型 |
| ------ | ------ |
| 国际版 | GPT-5.2、Gemini-3.1-Pro、Claude-Sonnet-4.6、DeepSeek-V3.2、Doubao-Seed-1.8、Kimi-K2.5、Qwen3.5-Plus、GLM-5、MiniMax-M2.5 | 
| 国内版 | GLM-5、DeepSeek-V3.2、Doubao-Seed-1.8、Kimi-K2.5、Qwen3.5-Plus、MiniMax-M2.5、LongCat| 

搜索引擎：
- 国际版支持 Google、Microsoft Bing 等多种搜索引擎
- 国内版支持 Microsoft Bing、百度等多种搜索引擎。

支持的操作系统：Tabbit 目前支持 macOS 12.0 及以上和 Windows 10/11 系统。macOS 版本目前仅支持搭载 Apple 芯片的设备。

重点是 Tabbit 目前处于测试阶段，核心功能均可免费使用。所以如果想薅国外三巨头羊毛的话，可以下一个海外版。

## 标签管理

接下来看看标签管理，一般浏览器要么横向标签，要么纵向标签，Tabbit 让你自己任意选择。

默认我横向打开了很多标签，然后点击智能管理标签

![ezimage-1772546989326.png](https://images.flashnote.top/2026/03/1772546989559-15el36lw.png)

再选择纵向排列

![ezimage-1772547316257.png](https://images.flashnote.top/2026/03/1772547316550-8lqjq227.png)

它并没有按域名之类的分类，而是根据内容来分类

![ezimage-1772547432106.png](https://images.flashnote.top/2026/03/1772547432296-c05ejrog.png)

页面上我打开了我的个人主页和同域名下的其他 3 个字域名，，前面被分在了 AI Agents 目录，EzImage，我制作的一个图床 vscode 插件被分在了 Content Createion Tools 里，我觉得还是蛮准确的。

![ezimage-1772548154665.png](https://images.flashnote.top/2026/03/1772548154905-rt3wd6aj.png)

点进标签组是一个卡片视图，每个页面都有缩略图预览，一眼看到内容。还有个「收藏全部」按钮，一键把整组页面存下来。

![ezimage-1772548116614.png](https://images.flashnote.top/2026/03/1772548117014-87ngnn9l.png)

纵向排列的好处是，就像目录一样，层级分明，也不用担心，标签排不下了。

之前就是因为标签太多，有时我会选择把一个 chrome 浏览器，分成多个，现在不会出现这种情况了。

但是话说回来，智能标签管理也有一个问题，就是只能智能管理，不能手动管理，比如我想把我的域名下的所有页面，单独放在一个标签下，或者现在我会打开多个 github 页面，想把他们放在一个 Github标签下，目前好像都无法自定义，希望后续能支持。

## 妙招

还记得我们前面说到的 通过 **/** 调出妙招吗？那么如何创建自己的妙招呢？

![ezimage-1772586222857.png](https://images.flashnote.top/2026/03/1772586223097-4j96hgby.png)

点击灯泡，点击 All Skills 进到 skills 管理页面

![ezimage-1772586320735.png](https://images.flashnote.top/2026/03/1772586320977-6wiy0k8k.png)

然后点击 Creat Skill 或者直接点击灯泡页面的Create Skill 按钮，都可以进到创建妙招的界面。

![ezimage-1772586513256.png](https://images.flashnote.top/2026/03/1772586513635-p5ns8nyz.png)

可以看到 Skill 分为三类：

- Prompt(提示词)
- Script(脚本)
- Agent(智能体)

接下来，我们一个一个演示一下，

### Prompt 妙招
就拿 Tabbit 官网演示，创建一个 Prompt

![ezimage-1772587732292.png](https://images.flashnote.top/2026/03/1772587732570-xlqbe2yt.png)

发送调用，总结的没毛病。

![ezimage-1772595668781.png](https://images.flashnote.top/2026/03/1772595669113-9sh7hqrv.png)

简单一句话，方便后续调用，你说直接对话框输入不香吗？OK, 当然没问题，愿意多打字，也没关系。


这只是一个极其简单的演示，如果你有更复杂的需求，需要写大几百字的提示词，然后还经常使用到， 你的选择是什么呢？

### Script

脚本妙招，可玩性就更多了

![ezimage-1772588026297.png](https://images.flashnote.top/2026/03/1772588026675-pdc2999z.png)

可以自己写脚本，可以通过对话让AI帮你生成。
可以设置成打开网页自动运行
可以在所有网页生效，你也可以选择指定网页生效


嗯？这是不是就是浏览器插件了，而且嘴说就行了。

直接整一个

![ezimage-1772588787049.png](https://images.flashnote.top/2026/03/1772588787455-bb2s0ziv.png)

命个名mp3，点击保存

![ezimage-1772589050879.png](https://images.flashnote.top/2026/03/1772589051334-psi7iy5q.png)

然后直接使用, 输入 /mp3 就行了

![ezimage-1772589125182.png](https://images.flashnote.top/2026/03/1772589125596-7exeul5g.png)


一会儿，结果就出来了，但是第一版给的在线的都无法播放，就先尝试添加本地音乐吧！

然后让它又修改了一版，给我添加了本地音乐支持功能。

果然，在线的还是播放不了，直接本地下载吧！

哎，这年头，谁还下载音乐到本地啊，为了测试，找了半天，好在音乐是正常播放的。

![ezimage-1772590487076.png](https://images.flashnote.top/2026/03/1772590487422-yvaxuse6.png)

下一步可以继续优化这个 mp3 播放器，或者做个mp4, 当然，你也可以发挥你的想象力，做更多有意思的脚本。

### Agent
Agent 妙招是其中最强大的一种。支持变量模板，比如设置「帮我采集一下{{公众号后台链接}}的前 10 条数据包括字段标题、摘要、链接、发布时间」，下次运行就可以动态根据当前时间的最新 10 条数据了。

![ezimage-1772590741031.png](https://images.flashnote.top/2026/03/1772590741246-wjrdj0z6.png)


编辑完后，点击保存，然后就可以使用了

![ezimage-1772591355244.png](https://images.flashnote.top/2026/03/1772591356146-zhs7trxs.png)

说一下调用 Agent 能力，都做了哪些事，
- 菜单栏多了一个 Task:采集公众号数据 页签
- 中间内容区域被自动控制
- 右侧显示任务进度

最终结果：

## Agent

其实，在上一步介绍妙招的时候，我们使用 Agent 妙招，就已经默认开启了 Agent 能力，就不多介绍了。

一句话总结一下：它能理解需求，拆分任务，访问网站、填写表单、执行操作，最后交付结果。

我们要做的就是想清楚要做啥，下发精准指令，接下来泡杯茶，静观它执行，等待结果输出就行了。
